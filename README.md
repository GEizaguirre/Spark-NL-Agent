# Spark AI Agent

This is brief demonstration code of the LangChain SparkSQL toolkit. Ask questions in plain English and get SQL queries executed on Spark.

## What is This Project?

This project implementes an AI-powered agent that:
1. Takes a **natural language question** (e.g., "What are the names of all superheroes who can fly?")
2. Uses an **LLM (Large Language Model)** to understand the question and generate SparkSQL
3. Executes the query on **Apache Spark** and returns the results

## Key Technologies

### LangChain / LangGraph

**LangChain** is a framework for building applications powered by language models. It provides:
- Standardized interfaces to connect with different LLMs (OpenAI, Google Gemini, Anthropic Claude, etc.).
- Tools and utilities for building complex AI workflows.
- Memory and state management for conversational agents.

**LangGraph** extends LangChain to build stateful, multi-actor applications. In this project, it orchestrates a **Reasoning and Action (ReAct) agent** that:
1. Reasons about what information it needs.
2. Takes actions (listing tables, checking schemas, running queries).
3. Observes the results and decides next steps.

### Apache Spark

**Apache Spark** is a distributed data processing engine designed for large-scale data analytics. Key features for this project:
- **SparkSQL**: SQL interface for querying structured data.
- **Distributed Computing**: Processes data across multiple nodes
- **In-Memory Processing**: Fast execution by keeping data in RAM.

In this project, Spark acts as the query execution engine that runs the SQL generated by the LLM. It runs locally, but it could be extended to distributed, real-scale Spark clusters.

## Sample Databases

The `db/` folder includes sample databases from the [BIRD Text-to-SQL Benchmark](https://bird-bench.github.io/) for testing. These SQLite databases cover various domains (superhero, financial, formula_1, etc.) and are used to evaluate the agent's ability to translate natural language questions into correct SQL queries.

## Setup

### 1. Prerequisites

- Python 3.10+
- Java 11+ (required for Spark)

### 2. Create Virtual Environment

```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure API Keys

Copy the template and add your Google AI Studio API key:

```bash
cp .env.template .env
```

[Here's a guide on how to get you Google AI Studio API Key](https://ai.google.dev/gemini-api/d).

Edit `.env` and add your LLM provider key:

```
GOOGLE_API_KEY=your_google_api_key_here
```

This project is also compatible with Claude, OpenAI and Cloudflare API Keys.

### 5. Run the Example

```bash
python test_nl_query.py
```

## How It Works

1. **Database Loading**: SQLite databases are loaded into Spark as temporary views
2. **Agent Initialization**: A ReAct agent is created with tools for:
   - `list_tables_sql_db`: List available tables
   - `schema_sql_db`: Get table schemas
   - `query_checker_sql_db`: Validate SQL syntax
   - `query_sql_db`: Execute SparkSQL queries
3. **Query Processing**: The agent reasons through the question, explores the schema, and generates SQL
4. **Execution**: The generated SQL runs on Spark and results are returned

## Supported LLM Providers

- **Google Gemini** (default)
- **OpenAI GPT**
- **Anthropic Claude**
- **Cloudflare Workers AI**

Change the provider in `test_nl_query.py`:

```python
llm = get_llm(provider=Provider.OPENAI.value)  # or CLAUDE, CLOUDFLARE
```
